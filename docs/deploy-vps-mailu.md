# Deploy Mail Kit With Mailu On A VPS

This guide uses Mailu as the hidden mail engine and Mail Kit as the product UI/API.

## Target Architecture

```txt
app.example.com  -> Mail Kit web
api.example.com  -> Mail Kit API
mail.example.com -> Mailu SMTP/IMAP/mail engine
```

Mail Kit owns users, domains, API keys, DNS verification, logs, inbox UI, and admin controls.
Mailu owns SMTP, IMAP, DKIM signing, spam filtering, and mailbox storage.

## VPS Requirements

- Public static IPv4.
- Docker and Docker Compose.
- Open ports: `25`, `465`, `587`, `993`, `80`, `443`.
- Reverse DNS / PTR set by the VPS provider:

```txt
VPS_IP -> mail.example.com
```

Avoid a VPS provider that blocks outbound port `25`.

## DNS For The Platform Domain

For the domain that runs Mail Kit and Mailu:

```txt
A     app.example.com    VPS_IP
A     api.example.com    VPS_IP
A     mail.example.com   VPS_IP
MX    example.com        mail.example.com
TXT   example.com        v=spf1 mx ~all
TXT   _dmarc.example.com v=DMARC1; p=none
```

Add the DKIM TXT record generated by Mailu after the mail domain is created.

## Mailu Setup

The safest long-term path is still to compare this repo's compose file with the official Mailu setup utility output when you upgrade Mailu. For the first Mail Kit VPS proof, this repo includes a Mailu engine setup:

```bash
DOMAIN=example.com \
MAIL_HOST=mail.example.com \
POSTMASTER=postmaster \
MAILU_ADMIN_PASSWORD='strong password' \
./scripts/mailu-setup.sh
```

That command:

- generates `deploy/mailu.env` if it does not exist
- creates persistent Mailu directories under `/opt/mailu`
- starts `docker.maulu-setup.yml`
- creates the first Mailu admin account when `MAILU_ADMIN_PASSWORD` is set
- prints the Mail Kit server env values needed for SMTP, IMAP, and Mailu API integration

Important: `docker.maulu-setup.yml` is a standalone Mailu setup and binds public ports `80` and `443` for Mailu's HTTP API/admin surface and certificate handling. Do not run another public reverse proxy on `80/443` at the same time unless you intentionally adapt the reverse proxy layout.

Mailu values to enable for Mail Kit integration:

```env
API=true
WEB_API=/api
API_TOKEN=replace-with-a-long-random-token
```

Mail Kit expects Mailu to provide:

```txt
SMTP submission: mail.example.com:587
IMAP over TLS:   mail.example.com:993
API URL:         https://mail.example.com/api
```

Keep Mailu admin/webmail private or operator-only. End users should only see Mail Kit.

Never delete `/opt/mailu` unless you intentionally want to wipe Mailu mailbox data, DKIM keys, and stored mail.

## Mail Kit Production Setup

Copy and fill the example env files:

```bash
cp .env.production.example .env.production
cp apps/server/.env.production.example apps/server/.env.production
cp apps/web/.env.production.example apps/web/.env.production
```

Important values:

```env
APP_DOMAIN=app.example.com
API_DOMAIN=api.example.com
PUBLIC_API_URL=https://api.example.com

BETTER_AUTH_URL=https://api.example.com
CORS_ORIGIN=https://app.example.com

SMTP_HOST=mail.example.com
SMTP_PORT=587
EMAIL=postmaster@example.com
EMAIL_PASSWORD=MAILU_MAILBOX_PASSWORD

MAIL_ENGINE_IMAP_HOST=mail.example.com
MAIL_ENGINE_IMAP_PORT=993
MAIL_ENGINE_MODE=mailu
MAIL_ENGINE_API_URL=https://mail.example.com/api
MAIL_ENGINE_API_TOKEN=MAILU_API_TOKEN
MAIL_ENGINE_SYNC_ENABLED=true
MAILBOX_SECRET_KEY=LONG_RANDOM_SECRET_FOR_MAILBOX_CREDENTIAL_ENCRYPTION
```

Start Mail Kit:

```bash
docker compose --env-file .env.production -f docker-compose.production.yml up -d --build
```

Run database setup from the server container:

```bash
docker compose --env-file .env.production -f docker-compose.production.yml exec server bun run db:migrate
docker compose --env-file .env.production -f docker-compose.production.yml exec server bun run db:seed
```

## VPS Readiness Check

Run the checker from the VPS after Mailu and Mail Kit are started:

```bash
DOMAIN=example.com \
MAIL_HOST=mail.example.com \
APP_HOST=app.example.com \
API_HOST=api.example.com \
VPS_IP=1.2.3.4 \
DKIM_SELECTOR=dkim \
MAILKIT_ENV=apps/server/.env.production \
MAILU_ENV=deploy/mailu.env \
./scripts/check-vps-mailu.sh
```

The script prints what is present and what is missing for:

- Mailu containers and API settings.
- Mail Kit Mailu/SMTP/IMAP env values.
- TCP ports `25`, `465`, `587`, `993`, `80`, and `443`.
- `A`, `MX`, `SPF`, `DKIM`, and `DMARC` DNS records.
- VPS PTR / reverse DNS.
- Mailu API reachability.

It cannot fully prove Gmail/Outlook deliverability by itself. For that, send a real message from Mail Kit, open the delivered message headers in Gmail/Outlook, and confirm SPF, DKIM, and DMARC all pass.

## First Sending Test

1. Create a domain in Mail Kit.
2. Confirm the domain engine status is `provisioned` or `manual` if Mailu API automation is disabled.
3. Add the DNS records shown in Mail Kit.
4. Copy the DKIM value from Mailu if Mailu generated it.
5. Verify the domain in Mail Kit.
6. Create an API key.
7. Send with `POST /emails/send`.
8. Confirm the message arrives in Gmail/Outlook.
9. Inspect message headers and confirm SPF, DKIM, and DMARC pass.

## First Receiving Test

1. Create a mailbox in Mail Kit such as `hello@example.com`.
2. Confirm the mailbox engine status is `provisioned` or `manual`.
3. If using manual mode, create the same mailbox in Mailu and put its IMAP username/password in Mail Kit.
4. Send from Gmail to `hello@example.com`.
5. Confirm Mailu receives the message.
6. Click Sync on the Mail Kit mailbox or wait for the sync worker.
7. Confirm the message appears in Mail Kit Inbox.
8. Reply from Mail Kit and confirm delivery.

## Admin Checks

1. Open Admin -> Mail.
2. Confirm domains, mailboxes, outbound logs, inbound messages, and API key metadata are visible.
3. Trigger DNS verification.
4. Trigger domain provisioning sync.
5. Trigger mailbox sync.
6. Disable and re-enable a mailbox.
7. Suspend and unsuspend a domain.

## Troubleshooting

- If Gmail does not receive mail, check outbound port `25`, PTR, SPF, DKIM, DMARC, and IP reputation.
- If DNS verification fails, query records from a public resolver and wait for DNS propagation.
- If Mail Kit cannot provision Mailu, check `MAIL_ENGINE_API_URL`, `MAIL_ENGINE_API_TOKEN`, Mailu `API=true`, and Mailu logs.
- If inbound sync fails, check `MAIL_ENGINE_IMAP_HOST`, `MAIL_ENGINE_IMAP_PORT`, mailbox credentials, TLS, and Mailu IMAP logs.
- If auth cookies fail in production, check `BETTER_AUTH_URL`, `CORS_ORIGIN`, HTTPS, and cookie domain behavior.

## Notes

- Mail Kit does not expose Mailu to end users.
- DKIM signing is Mailu's job in this phase.
- Mail Kit stores imported messages for dashboard UX, but Mailu remains the real mail engine.
- If sending fails, check port `25`, PTR, SPF/DKIM/DMARC, and VPS IP reputation before assuming the app is broken.
